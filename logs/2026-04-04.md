### 学んだこと

#### Claudeとは何か
- Claudeは「有益で、正直で、無害」という3つの原則に基づいて設計されたAIシステム
- 単純なチャットボットではなく、業務の自動化と効率化を実現するパートナー的存在
- 主要な5つの機能:
  - ライティングとコンテンツ作成
  - 調査と分析
  - コーディング支援
  - 問題解決と推論
  - 新しいことを学ぶ
- アクセス方法は4つ: Claude.ai / Claude Code / Claude and Slack / Claude for Excel

#### プロンプト作成の3ステップ
1. **状況説明** — 自分の役割・目標・背景情報を伝える
2. **タスクの定義** — 書いてほしいのか、分析か、構築か、具体的な行動を指示する
3. **ルールの指定** — スタイル・トーン・サンプルを提示する

#### 4Dフレームワーク（AI活用能力）
- **Delegation（委任）** — 適切なタスクをAIに任せる
- **Description（記述）** — 明確な指示を書く
- **Discernment（識別）** — 出力の品質を判断する
- **Diligence（勤勉）** — 継続的に改善する

#### よくある課題と解決策（辞典として活用できる）
| 課題 | 原因 | 解決策 |
|------|------|--------|
| 回答が一般的すぎる | 背景情報不足 | 対象者・役割・制約を具体的に追加 |
| 回答が長すぎる/短すぎる | 長さを推測している | 「2段落で」「100語以内で」と明示 |
| フォーマットが違う | 提示方法が不明 | 例を示す or 構造を具体的に説明 |
| 情報が間違っている | もっともらしいが誤り | 重要な事実は独自検証、情報源を要求 |
| 口調が合わない | デフォルトのトーン | 「会話調で」「フォーマルに」と指定 |

#### ワークフローでのClaude評価方法
1. 普段の業務の事例を5〜10個集める
2. 同様の出力を得るプロンプトを作成する
3. Claudeの出力と自分の例を比較する
4. 指示の調整・例の追加・人間レビュー箇所の特定で改善する
